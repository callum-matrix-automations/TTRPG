Ok, I'm going to address all of those items.

Tier 1:

## Spatial World and Movement

I want an actual map yes, but I don't want it entirely pre-made. I want more of something where I define the skeleton 
by giving the LLM different hub locations. For example "Residential" "Commercial" etc. The AI can then define "spokes" 
of those hubs, and those spokes can have further spokes within them. This allows there to be some novelty to every playthrough.

I also want NPCs to be able to travel between these places, the AI LLM that controls them should know what it is able to do. This is only for NPCs in the scene though, when off-screen they don't decide (that would be the narrator/GM deciding).

Triggers that occur between travel should probably be random generated in-code, so an event can be triggered then the GM decides what actually will happen.

And yes, some places are locked, need reputation etc. 


## Turn/Phase structure

This sounds good, travelling between zones should cost a pre-defined and predictable unit of time, and at the end of a scene the GM can decide how much time passed.

Scenes will be triggered upon entering a location and engaging with someone. There should be a chance of a random event occuring when entering a location too, which would trigger a scene without the player deciding to trigger it


## Resource Management

We don't need to be granular on food and rations. I don't want to get too granular on this by addingn encumbrance. The only resource I want would be HP, energy, and gold.

## Rest & Recovery system

Yes this is something we should have, travel should cost some energy (not much), energy used can be determined post-scene by the GM
